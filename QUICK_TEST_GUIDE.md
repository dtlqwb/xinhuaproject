# 部署后快速测试指南

**测试时间:** 约15-20分钟  

---

## 🚀 快速测试流程

### 步骤1: 验证部署状态 (2分钟)

#### 在服务器上执行:
```bash
# SSH登录服务器
ssh ubuntu@您的服务器IP

# 检查Docker容器状态
cd xinhuaproject
docker-compose ps
```

**预期结果:**
```
NAME                STATUS              PORTS
sales-mysql         Up                  3306/tcp
sales-backend       Up                  0.0.0.0:8080->8080/tcp
sales-frontend      Up                  0.0.0.0:80->80/tcp
admin-frontend      Up                  0.0.0.0:81->80/tcp
```

**如果所有容器都是"Up"状态,继续下一步** ✅

---

### 步骤4: 测试后端API (5分钟)

#### 使用浏览器开发者工具或Postman:

**4.1 测试登录API**
```bash
curl -X POST http://82.156.165.194:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"test","password":"123456"}'
```

**预期响应:**
```json
{
  "code": 200,
  "data": {
    "token": "eyJhbGci...",
    "userInfo": {...}
  }
}
```

---

**4.2 测试添加客户(FormData)**
```bash
curl -X POST http://82.156.165.194:8080/api/customer/add \
  -F "content=李四,XYZ公司,总监,13900139000,需要CRM系统" \
  -F "salesId=1"
```

**预期响应:**
```json
{
  "code": 200,
  "data": {
    "id": 2,
    "name": "李四",
    "company": "XYZ公司",
    ...
  }
}
```

---

**4.3 测试获取客户列表**
```bash
curl http://82.156.165.194:8080/api/customer/list?salesId=1
```

**预期响应:**
```json
{
  "code": 200,

    {
      "id": 1,

      ...
    },
```

### 步骤6: 性能检查 (2分钟)

#### 6.1 检查资源使用
```bash
# 查看Docker容器资源使用
docker stats --no-stream

# 查看服务器资源
free -h
df -h
```

**预期:**
- CPU使用率 < 50%
- 内存使用率 < 70%
- 磁盘空间充足

---

#### 6.2 检查响应时间

在浏览器中:
1. 打开开发者工具 (F12)
2. 切换到Network标签
3. 刷新页面
4. 查看加载时间

**预期:**
- 页面加载 < 3秒
- API响应 < 1秒
- AI生成 < 5秒

## 📞 获取帮助

### 查看日志
```bash
# 后端日志
docker-compose logs -f backend

# 前端日志
docker-compose logs -f sales-frontend
docker-compose logs -f admin-frontend

# 数据库日志
docker-compose logs -f mysql
```

### 重启服务
```bash
# 重启单个服务
docker-compose restart backend

# 重启所有服务
docker-compose restart
```

### 查看状态
```bash
# 容器状态
docker-compose ps

# 资源使用
docker stats

# 磁盘空间
df -h
```
