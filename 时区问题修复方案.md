# 客户信息收集系统 - 时区问题修复方案

> 本文档列出所有时区相关问题和具体修改步骤，请按顺序执行。
> 项目路径：`D:\360MoveData\Users\wangbo\Desktop\客户信息收集\xinhuaproject`

---

## 一、问题根因

Docker 容器默认使用 UTC 时区，项目面向中国用户应使用 Asia/Shanghai (UTC+8)。当前三个关键容器（MySQL、Java后端、前端nginx）**均未设置时区**，导致：

1. JVM 的 `LocalDateTime.now()` 返回 UTC 时间，比北京时间晚8小时
2. MySQL 的 `CURRENT_TIMESTAMP` 写入 UTC 时间
3. Jackson JSON 序列化日期时使用 UTC 时区
4. Dashboard"今日新增"等统计在凌晨0-8点查的是错误日期的数据

## 二、必须修复（🔴 高优先级）

### 2.1 修复 JVM 时区

**文件**: `backend/Dockerfile`

**第9行**，修改前：
```dockerfile
ENV JAVA_OPTS="-Xms512m -Xmx1024m -Dfile.encoding=UTF-8"
```

修改后：
```dockerfile
ENV JAVA_OPTS="-Xms512m -Xmx1024m -Dfile.encoding=UTF-8 -Duser.timezone=Asia/Shanghai"
```

> 说明：使 JVM 的 `LocalDateTime.now()`、`LocalDate.now()` 等调用返回北京时间。

---

### 2.2 修复 MySQL 服务器时区

**文件**: `docker-compose.yml`

**第14行**，修改前：
```yaml
    command: --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci
```

修改后：
```yaml
    command: --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci --default-time-zone='+08:00'
```

同时给 mysql 容器加 TZ 环境变量。在第7行 `MYSQL_ROOT_PASSWORD` 之后加一行：

修改前：
```yaml
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_PASSWORD:-root123}
      MYSQL_DATABASE: sales_customer_db
```

修改后：
```yaml
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_PASSWORD:-root123}
      MYSQL_DATABASE: sales_customer_db
      TZ: Asia/Shanghai
```

> 说明：使 MySQL 的 `CURRENT_TIMESTAMP`、`NOW()` 等函数返回北京时间，与 JDBC 的 `serverTimezone=Asia/Shanghai` 保持一致。

## 三、建议修复（🟡 中优先级）

### 3.1 Spring Jackson 时区配置

**文件**: `backend/src/main/resources/application.yml`

在 `spring:` 下、`datasource:` 之前加 Jackson 配置：

修改前：
```yaml
spring:
  datasource:
```

修改后：
```yaml
spring:
  jackson:
    time-zone: Asia/Shanghai
    date-format: yyyy-MM-dd HH:mm:ss
  datasource:
```

同样在 `spring:` 下、`datasource:` 之前加：

修改前：


```yaml

  jackson:
    time-zone: Asia/Shanghai
```

### 3.2 实体类日期字段加 @JsonFormat 注解

## 四、可选优化（🟢 低优先级，本次可不改）

### 4.1 SQL DATETIME → TIMESTAMP

当前 `schema.sql` 所有时间列使用 `DATETIME` 类型。修复 MySQL 时区后 `DATETIME` 也能正常工作，保持现状即可。如果未来需要跨时区部署，可改为 `TIMESTAMP` 类型。

### 4.2 前端日期处理

修复后端时区后，前端现有代码无需改动：
- `Customers.vue` 的 `new Date(time).toLocaleString('zh-CN')` — 依赖浏览器时区，后端返回正确时间即可
- `Reports.vue` 的 `dateTime.replace('T', ' ').substring(0, 16)` — 加了 Jackson date-format 后不再有 `T`
- `CustomerCard.vue` 的时间格式化 — 同理

## 五、修改完成后验证步骤

1. **重新构建并启动容器**：
   ```bash
   docker-compose down
   docker-compose build --no-cache backend
   docker-compose up -d
   ```

2. **验证 MySQL 时区**：
   ```bash
   docker exec sales-mysql mysql -uroot -proot123 -e "SELECT @@global.time_zone, @@session.time_zone, NOW();"
   ```
   期望输出：`+08:00` 和当前北京时间

3. **验证 JVM 时区**：
   ```bash
   docker exec sales-backend sh -c "date && echo '---' && java -cp app.jar -Dloader.main=java.lang.System org.springframework.boot.loader.JarLauncher 2>/dev/null || date"
   ```
   或直接查看日志时间戳是否为北京时间

4. **功能验证**：
   - 在管理端 Dashboard 查看"今日新增客户"数字是否正确
   - 添加一个客户，检查 `create_time` 是否为北京时间
   - 生成日报，检查日期是否为当天
   - 导出 Excel，检查时间列是否为北京时间
